# Gouvernance

Fulgurite est maintenu comme projet source-available avec collaboration publique, direction produit centralisée et droits commerciaux réservés.

## Modèle De Projet

Les mainteneurs sont responsables de :

- décisions de roadmap
- gestion des releases
- politique de sécurité
- modération du dépôt
- direction architecturale
- politique de licence et commerciale
- acceptation, refus ou réorientation des contributions

La roadmap actuelle est décrite dans [ROADMAP.md](ROADMAP.md).

## Décisions

Les mainteneurs peuvent accepter, refuser, différer ou demander des changements selon :

- adéquation produit
- impact sécurité
- maintenabilité
- cohérence architecturale
- clarté légale
- charge de support
- qualité de documentation

Les changements majeurs doivent être discutés avant implémentation.

## Droits Des Contributeurs

Fulgurite suit un modèle protecteur :

- les contributeurs gardent la propriété du contenu original soumis
- les contributeurs restent responsables de l’origine et de la légalité de leurs contributions
- les contributeurs accordent les droits décrits dans [CONTRIBUTOR_TERMS.md](CONTRIBUTOR_TERMS.md)
- les fonctionnalités écrites par un contributeur restent dans le dépôt public source-available sauf accord écrit contraire

## Limites Commerciales

Le propriétaire du projet peut construire des offres commerciales autour de Fulgurite : support payant, hébergement, licence commerciale, intégrations premium ou modules futurs.

Une fonctionnalité écrite par un contributeur ne sera pas déplacée dans une offre payante ou commerciale sans accord écrit préalable du contributeur.

## Langue Et Documentation

Les versions françaises de convenance sont dans [project/fr](README.md). La documentation utilisateur est dans [docs/fr](../../docs/fr/01-index.md).
