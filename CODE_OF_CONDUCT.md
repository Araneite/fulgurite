# Code of Conduct

Fulgurite should be a calm, respectful and technically serious place to collaborate.

## Expected Behavior

Participants are expected to:

- be respectful and constructive
- assume good intent before escalating
- critique ideas, code and decisions without attacking people
- welcome questions from contributors with different experience levels
- keep discussions focused on the work
- respect maintainer decisions and project boundaries

## Unacceptable Behavior

The following behavior is not acceptable:

- harassment, threats or intimidation
- discriminatory or hateful language
- personal attacks, mockery or humiliation
- doxxing or sharing private information without consent
- deliberate disruption, spam or bad-faith escalation
- publishing private security details before a fix or mitigation is available

## Maintainer Responsibilities

Maintainers may moderate discussions, close threads, remove content, decline contributions or restrict participation when needed to protect the project and its contributors.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers privately when possible.

If the incident involves a vulnerability or security-sensitive information, follow [SECURITY.md](SECURITY.md).

## Scope

This code of conduct applies to issues, pull requests, code review, discussions and other project spaces managed by the maintainers.
